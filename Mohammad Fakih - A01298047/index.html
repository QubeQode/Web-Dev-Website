HTML

HTML (HyperText Markup Language) is used to produce content for websites. It is capable of differentiating text into titles, headings, sub-headings and paragraphs. It can also create basic formats like producing numbered lists, bulleted lists and tables. In short it defines the basic layout and format of the content on a webpage.

How to Write HTML

HTML functions on elements which wrap content to make it appear or behave a certain way. The wrappers on either end are referred to as tags.

<opening tag> CONTENT BODY <closing tag>

The combination of all the above features is what is referred to as an element.

Elements can also contain extra information in the form of attributes.

<p class="content-descriptor"> CONTENT BODY <closing tag>

The above example features an attribute called class with a placeholder value of "content-descriptor".

Invoking an Attribute

The format must be as follows:

  1. There must be a space between the attribute name and the element name

  2. The attribute name must be followed by an = sign

  3. The attribute value must be wrapped in " "

Nesting Elements

Nesting refers to when an element is put inside another element. When nesting elements the tag last opened must be the first closed. Otherwise the browser attempts to guess your meaning which can result in an unpredictable output.

Empty Elements

Some elements are empty and do not wrap any content. An example of this is <img> 

                    
<img 
    src="image_directory/image_name.jpg" 
    alt="description in case img doesn't load"
>

As can be seen there is no closing tag for this element. This is as it does not wrap anything. Its purpose is simply to embed an image in the HTML page.

Anatomy of an HTML Doc 

                    
<!DOCTYPE html>
<html lang="en-US">

<head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width">
    <title>My test page<title>
</head>

<body>
    <img src="image/file/path" alt="image description">
</body>

</html>

<!DOCTYPE html> - ensure the document is recognized as an HTML doc

<html> </html> - wraps all the content in the page you are coding

<head> </head> - a container for all the content you want to introduce but don't want users to see

<meta charset="utf-8"> - enables doc to read and execute any textual content put in it

<meta name="viewport" content="width=device-width"> - ensures page renders at width of user's viewport

<title> </title> - sets the title of your page (tab name)

<body> </body> - contains all the content you want your user to see and interact with

Headings

  • <h1> - Title of webpage

  • <h2> - Title of Section

  • <h3> - Subheading 1

  • <h4> - Subheading 2

  • <h5> - Subheading 3

  • <h6> - Subheading 4

Paragraphs

Heading tags specify textual content that is to be treated as a heading or sub-heading. As the numbers increase in value, the size and boldness of the title formatting decreases.

Paragraphs also function exactly as they do in written english. The only difference being that they too must be wrapped in element tags. Specifically the element tag:

<p> PARA CONTENT <p>

Lists

There are two types of lists within HTML, the unordered and ordered list. The difference between them being that the unordered list produces bullet points. Whereas the ordered list produces descending numbers. Each of the two has their own tag.

<ul> </ul> - unordered list

<ol> </ol> - ordered list

Regardless of the type of list, there is a further, nested element tag that is required to change the list structure:

<li> </li>

The above tag flags each item within the list an can contain a paragraph tag wrapping text or even an anchor tag containing a hyperlink. The latter use case is common when building a nav-bar structure.

Links

The anchor tag is used to wrap any text that you wish to be a hyperlink. Using the href attribute we can create a reference point for where the hyperlink goes to, through the value assignment. A good way to remember this is that href stands for "Hypertext Reference".

<a href:"https://www.gotothissite.com"> CHECK OUT THIS SITE!!!! </a>

Always include the protocol (https:// or http://) or you may get unexpected results.

CSS

CSS (Cascading Style Sheets) primarily revolves around presentation. Although HTML includes some basic formatting of its own, CSS injects color, font styling and customizable spacing into designs. The easy to read, colorful and unique websites of today's web all make use of these exact principles. In particular, CSS's spacing capabilities are a large factor to the clean look of modern websites.

How to Write CSS

CSS functions on rulesets that dictate rules to the web browser on how to display given HTML elements. The two languages work in tandem in order to create web content customized to the creator's vision. As such the first thing that must be done is to connect the CSS file to the HTML file.

<link href="filePath/styles/fileName.css" rel="stylesheet">

The stylesheet should be made in a seperate directory to the HTML file called "styles". And the above code should be replicated in the head of the HTML document.

Rulesets 

                    
selector {
    property: property value;
}

The above code snippet is what is referred to as a ruleset. Every ruleset consists of the following pieces:

selector - Defines the element to be styled

declaration - umbrella term for the property and property value. The declaration contains the style rule for the HTML element(s) defined by the selector. Each declaration can only alter one property of an element. Though many declarations can be placed under one selector.

property - stylable feature in an element

propery value - defines the specific output/change in a given property

Selectors 

                    
selector1,
selector2,
selector3 {
    declaration;
}

As can be seen you can attribute multiple selectors to a singular ruleset, if you want your given rules to apply on a broader scope. Selectors come in many forms, with each form providing a unique specificity and universality.

element { - element selector - changes all elements in HTML file with the same tag

#id-name { - ID selector - changes all elements with an ID atrtibute of the same name in the HTML file.

.my-class { - class selector - changes all elements with a class attribute of the same name in the HTML file.

element[attribute="value"] { - attribute selector - changes all elements with a specific attribute and value in the HTML file.

element:pseudo-Class { - pseudo class selector - changes named element(s) when a specific condition is met

Font Declarations

color - change the color of a font

text-decoration - underline, strike through or overline font

font-size - establish the size of the font with the following measurements:

  • px - pixels
  • em - relative size where 1 is the size of the parent element
  • rem - relative size where 1 is the size of the root element in the HTML doc

font-style - italcize font

font-weight - bold font with varying degrees of boldness

text-transform - change font to uppercase, lowercase, capitalized or full-width

text-shadow: [horizontal offset] [vertical offset] [blur radius] [base color]

Font Family

The font-family is another font declaration. Though one that comes with specific considerations. Font must be chosen and declared carefully to ensure accessibility when designing a website.

Web Safe Fonts

  • Arial
  • Courier new
  • Georgia
  • Times New Roman
  • Trebuchet MS
  • Verdana

Default Fonts

  • serif - the letters contain floruishes
  • sans-serif - blocky flourish-less lettering
  • monospace - characters all have the same width
  • cursive - emulate handwriting
  • fantasy - decorative font

The font-family declaration contains a fallback system that will load a backup if the initial, requested font is not displayable by the browser. It is best practice to always start with the specific font(s) you would like and then the generic family as a fallback. For example:

font-family: Arial, Helvetica, sans-serif;

The above declaration asks for the Arial and Helvetica fonts from the sans-serif family. If neither of the two requested fonts are available it will at least display something similar from the generic root.

Text Layout Styles

text-align - align the text within the parent element

line-height - change the sutance between lines of text

letter-spacing - change the distance between letters

word-spacing - change the distance between words

text-underline-position - change the height of an underline

text-indent - define indentation of first line in each block of text

text-overflow - define how to signal that content is not being shown

word-break - whether or not to break words over two lines

CSS Box Model

Within CSS there are multiple facets that come together to produce a box or contianer. Moving from the inside out, they are as follows:

  1. content-box - the border around the information contained within the element or container
  2. padding - the spacing between the edge of the element and the information contained within
  3. border - the edge of the element
  4. margin - the spacing between the edge of the element and other elements

When we talk about spacing elements and shifting the layout in CSS it is largely done through altering these properties of an element.
